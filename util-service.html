UtilUserSearchService Class Methods

  • retrieveUserById()
    • Called by:
      • AddEdit.editUser()
      • ViewUser.retrieveUserById()
    • Params:
      • Business object (either UserForm or UserQuery)
    • Returns:
      • void
    • Purpose:
      • Sets data on a business object for a specific user based on ID. This will either need to be broken into 2 methods or have checks to see what values the passed in EBO needs since the 2 methods that call it require different data.
  • retrieveUserByQuery()
    • Called by:
      • ViewMany.retrieveUsersByQuery()
    • Params:
      • String representing search query
      • Integer for offset
      • Integer for limit
    • Returns:
      • ArrayList of UserListQuery EBOs
    • Purpose:
      • Create an ArrayList of UserListQuery EBOs based on a passed in string value to search on as well as an offset and limit of results to return (if paginating). This could be altered to have the parameter fields passed in as a new EBO that has those fields.
  • retrieveAllUsers()
    • Called by:
      • ViewMany.retrieveAllUsers()
    • Params:
      • Integer for offset
      • Integer for limit
    • Returns:
      • ArrayList of UserListQuery EBOs
    • Purpose:
      • Create an ArrayList of UserListQuery EBOs based on passed in values for limit and offset. This list will come from the list of all users. This will allow for pagination of the results.
  • updateUser()
    • Called by:
      • AddEdit.saveUser()
    • Params:
      • UserForm Business Object
    • Returns:
      • id of user to be used for display after database transaction has completed
    • Purpose:
      • Updates an existing user's data in the database. Returns the id of the user that will be used to display the result when requeried from an action redirect.
  • insertUser()
    • Called by:
      • AddEdit.saveUser()
    • Params:
      • UserForm Business Object
    • Returns:
      • id of user to be used for display after database transaction has completed
    • Purpose:
      • Adds a new user's data to the database. Returns the id of the user that will be used to display the result when requeried from an action redirect.
